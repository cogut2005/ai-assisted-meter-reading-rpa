# Output Files

## AI-assisted Meter Reading Validation RPA

This document defines the output files generated by the automation.

The automation produces structured CSV files and a short summary report. No image artifacts are generated.

---

## 1. Output Folder Structure

All generated files are stored under:

`data/output/`

Recommended structure:

```text
data/output/
├── processed_readings.csv
├── exceptions.csv
├── human_review_queue.csv
├── new_customer_candidates.csv
├── mail_drafts.csv
└── summary_report.txt
```

---

## 2. processed_readings.csv

This file contains only records that were successfully processed automatically.

Only valid non-AI records are included in this file. AI-based readings are not included because they require human review before any database update.

Purpose:

`processed_readings.csv` answers the question:

Which submissions were accepted and inserted into the database?

Columns:

```text
submission_id, customer_id, customer_name, meter_id, meter_number, meter_type, reading_date, previous_reading, new_reading, consumption_since_last, source, validation_status, database_update, notes
```

Example:

```csv
submission_id,customer_id,customer_name,meter_id,meter_number,meter_type,reading_date,previous_reading,new_reading,consumption_since_last,source,validation_status,database_update,notes
SUB-20260701-0001,1,Emma Carter,1,MTR-0001,electricity,2026-07-01,15605,15652,47,portal,processed,yes,Normal meter reading processed successfully.
SUB-20260701-0002,2,Liam Foster,2,MTR-0002,electricity,2026-07-01,9090,9175,85,mobile_app,processed,yes,Normal meter reading processed successfully.
```

Column meaning:

| Column | Meaning |
|---|---|
| submission_id | Internal ID generated by the automation |
| customer_id | Customer ID from the input CSV |
| customer_name | Customer name from the database |
| meter_id | Internal meter ID from the database |
| meter_number | Business-facing meter number |
| meter_type | Meter type, for example electricity or gas |
| reading_date | Submitted reading date |
| previous_reading | Latest previous reading for the same meter |
| new_reading | Accepted new meter reading |
| consumption_since_last | Calculated consumption since the previous reading |
| source | Submission source, for example portal or mobile_app |
| validation_status | Processing result, usually processed |
| database_update | Indicates whether the record was inserted into the database |
| notes | Short processing explanation |

## 3. mail_drafts.csv

This file contains mail drafts generated by the automation.

The automation does not send real emails automatically. It only creates structured mail drafts that can be reviewed before sending.

Purpose:

`mail_drafts.csv` answers the question:

Which communication should be prepared after processing?

Columns:

```text
submission_id, draft_type, recipient_type, recipient, subject, body, status
```

Example:

```csv
submission_id,draft_type,recipient_type,recipient,subject,body,status
SUB-20260701-0001,customer_confirmation,customer,customer_1@example.com,Meter reading received,"Dear customer, your meter reading for meter MTR-0001 on 2026-07-01 has been received and processed successfully. The recorded reading is 15652 kWh. Best regards, Customer Service Team",draft_created
SUB-20260701-0003,internal_review,review_team,meter-review@example.com,Meter reading requires manual review,"Submission SUB-20260701-0003 requires manual review. Reason: AI image-based reading. Meter: MTR-0003. Reading date: 2026-07-01. Please verify the extracted value before database update.",draft_created
SUB-20260701-0004,customer_correction,customer,customer_2@example.com,Meter reading correction required,"Dear customer, your submitted meter reading could not be processed because the reading value is missing or invalid. Please submit the reading again or upload a clear meter image. Best regards, Customer Service Team",draft_created
```

Column meaning:

| Column | Meaning |
|---|---|
| submission_id | Related internal submission ID |
| draft_type | Type of mail draft |
| recipient_type | Customer or internal review team |
| recipient | Recipient email address or placeholder |
| subject | Mail subject |
| body | Mail body |
| status | Draft status, usually draft_created |

Allowed draft_type values:

- customer_confirmation
- customer_correction
- internal_review

When to create each draft:

| Situation | Draft Type |
|---|---|
| Valid non-AI reading processed successfully | customer_confirmation |
| Customer input is missing or invalid | customer_correction |
| Case requires manual review | internal_review |
| AI image-based reading | internal_review |
| Consumption anomaly | internal_review |
| Reading lower than previous reading | internal_review |

## 4. summary_report.txt

This file contains a short run summary.

UiPath can read this file and show it in a Message Box after the automation finishes.

Purpose:

`summary_report.txt` answers the question:

What happened during this automation run?

Recommended format:

```text
AI-assisted Meter Reading Validation RPA
=======================================

Run timestamp: 2026-07-01 14:35:22

Input file:
data/input/portal_export.csv

Database:
data/history/master.db

Processing Summary
------------------
Total submissions: 8
Processed successfully: 2
Human review required: 3
Exceptions: 2
New customer candidates: 1
Database records inserted: 2
Image-based cases routed to review: 1
Mail drafts created: 6

Generated Output Files
----------------------
processed_readings.csv
exceptions.csv
human_review_queue.csv
new_customer_candidates.csv
mail_drafts.csv
summary_report.txt

Automation Status
-----------------
Completed successfully
```

Key notes:

- Valid non-AI readings were inserted into `meter_reading_history`.
- AI-based image readings were not inserted into the database.
- Records with anomalies or uncertain data were routed to human review.
- Hard validation errors were written to `exceptions.csv`.

Important rule:

The summary report should stay short.

Detailed row-level results belong in the CSV files, not in `summary_report.txt`.

---

## 5. Output Responsibility Overview

| Output | Main Purpose |
|---|---|
| processed_readings.csv | Shows records that were automatically accepted and inserted into the database |
| exceptions.csv | Shows records with hard validation errors |
| human_review_queue.csv | Shows records that require manual review |
| new_customer_candidates.csv | Shows unknown customers that need manual review |
| mail_drafts.csv | Shows generated customer and internal mail drafts |
| summary_report.txt | Shows a short run summary for UiPath |

---

## 6. Design Principle

CSV files contain structured processing results.

TXT report contains the run summary.

Mail drafts are created but not sent automatically.
